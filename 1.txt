Бизнес хочет, чтобы при "успешной выплате" клиенту автоматически отправлялась коммуникация (пуш в АМ).
Под «успешной выплатой» будем считать (также как для VOC) момент перехода анкеты в:
- статус 7 – Другу отправляем;
- статус 10 – Приглашающему отправляем.
Ограничение: Если у клиента с первого раза не прошла выплата на расчетный счет (из-за проблем со счетом) – он получит оповещение, хотя денег ещё не будет выплачено. Актуально для  менее  чем 1% клиентов.

Пуш (каждый текст - настройки после доработки):
- заголовок (обязательное)
- текст\подзаголовок (обязательное, max 254 символа)
- диплинк (опциональное) , в теории перечень существующих: https://confluence.moscow.alfaintra.net/pages/viewpage.action?spaceKey=AM&title=Deeplinks

------------- БФТ
1. Делаем только для физлиц. Физлица определяются - ориентировочно "все не U-Pin" (с разработчиком обсудить конкретную реализацию).
На Юриков сейчас технически невозможно делать нотификации (у нас нет X-pin, на которые надо отправлять).

2. В тексте нотификации необходима возможность использования данных:
- сумма "выплаты";
- дата оформления анкеты;
- название предложения = название правила (то же название, что отображается клиенту по анкете в истории рекомендаций).

3. Для каждого предложения должна быть возможность включить\выключить отправку нотификаций.
Отключаем\включаем целиком на Правило (и для друга, и для приглашающего одновременно).

4. Для каждого предложения должна быть возможность настроить разные тексты оповещения для друга и приглашающего по «выплате».

------ По согласованию с разработчиком и аналитиком договорились, что:
5. Тексты уведомлений (параметризованные) хранятся в MIS.
6. Одно уведомления содержит в себе тексты и для Друга, и для Приглашающего (делим на уровне Статуса, где Статус = статус анкеты 7 (выплата другу) или 10 (выплата приглашающему).
7. Делаем пока только для денежных выплат. Для альтернативных\подписок - будет обсуждаться в своё время в других задачах.
6. В UI на уровне Правила выводится выпадающий справочник доступных уведомлений:
	- только уведомления, настроенные для конкретного профиля (на проде заведен профиль №5, на деве\инте\прелайве будет скорее всего другой ИД профиля);
	- выводим уведомления с любым статусом:
		- и активные, и не активные (это для того, чтобы в Правилах не исчезли настроенные, если их переведут в неактивные);
		- если у уведомления статус "не активное" - отображать текст красным цветом в UI;
	- в выпадающем справочнике показывать только ИД уведомления + название;
		- после выбора (в Правиле) - отображать ИД + название + тексты уведомления:
			- если "статус 7" писать: Другу: <тут текст уведомления>;
			- если "статус 10" писать: Приглашающему: <тут текст уведомления>;
7. "Снепшот" хранить не требуется. Уведомление для Правила всегда актуально то, что настроено прямо сейчас, когда бы анкета для него не оформлялась.
